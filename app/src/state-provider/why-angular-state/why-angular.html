Why Choose AngularJS?

  • Reusability: the ability to create custom components through directives such as a main-nav or a login-form allows you to reuse these throughout your projects
  • Already embraces things that we already know such as HTML, CSS and JavaScript
  • Two-way data binding along with a lot of built in directives that solve issues we have been banging our head over using jQuery.
  • Simple dependency injection allowing the user to use other directives, services, and factories such as a router provider or animation services.
  • Built with testability in mind such as E2E and Unit Testing
  • Great for building small single page Apps
  • Easy to learn
  • Job opportunities won't disapear once Angular 2 comes out. A lot of companies are currently using version 1 and we still don't know when Angular 2 is even going to be released.
  • Check out all these web applications that are using Angular 1 Made With Angular

What you should not use Angular for

  • Building large web applications
  • If you think you have to know it in order to learn Angular 2 although it can't hurt
  • Gaming Applications
