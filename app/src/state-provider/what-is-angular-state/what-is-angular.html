What Is AngularJS

Developed by Google in 2009. It is simply JavaScript, which makes it easily compatible with your desktop and mobile browsers. It is a web application framework that follows the MVVM design pattern. Depending on the devloper, it can also be considered a MVC pattern. For those of you who don't know, MVVM stands for Model-View-View-Model.

  • Model: Responsible for maintaining the application data, a model in AngularJS is what allows your application to share state or attach data within your controllers.
  • View: It is what the user sees. AngularJS uses HTML as its templating language. View is bound to your $scope. Displays the data based what is supplied by the controller.
  • View Model: It is the place where we put our applications logic. Our application model lives within our controllers in AngularJS. The model view is what connects our model and view by controlling the state of the view and manipulating the model based on user interactions.

Framework Vs. Library

    Framework
  • Controls your application
  • Normally calls on your code to perform certain tasks based on the code that you have written.
  • Gives your code structure
  • You are not in charge but rather plug in your code
  • Can sometimes encapsulate useful libraries
  • Examples: ReactJS, Ember, Backbone and AngularJS
    Library
  • Packages of code where your application's code calls on the packages to perform certain tasks.
  • Acts as a toolkit
  • You are in charge
  • jQuery is a very popular library that works a lot with DOM manipulation
